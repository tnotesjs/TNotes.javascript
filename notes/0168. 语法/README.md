## 2. 🫧 评价

语法规则不难，但它决定了代码会不会被正确解析。这里最值得养成习惯的是：标识符命名清楚、语句结尾稳定、控制结构总是加代码块。

## 3. 🤔 大小写在 ECMAScript 中有什么影响？

ECMAScript 中的变量名、函数名、属性名、关键字和操作符都区分大小写。

这意味着 `test` 和 `Test` 是两个不同的标识符：

```js
let test = 1
let Test = 2

console.log(test) // 1
console.log(Test) // 2
```

同理，`typeof` 是关键字，不能拿来当函数名；但 `Typeof` 只是普通标识符，从语法上看是可以使用的。

实际写代码时，大小写差异最好只用于明确的命名约定，不要用来制造两个容易混淆的变量名。否则阅读和调试都会很吃力。

## 4. 🤔 标识符应该怎样命名？

标识符就是变量、函数、属性或参数的名称。

基本规则是：

- 第一个字符必须是字母、下划线 `_` 或美元符号 `$`。
- 后续字符可以是字母、下划线、美元符号或数字。
- 不能使用关键字、保留字、`true`、`false` 和 `null`。

ECMAScript 允许在标识符里使用某些扩展字符和 Unicode 字符，但日常开发中不推荐这样做。最常见、也最容易协作的风格是驼峰命名：

```js
let firstName = 'Ada'
let currentUser = { name: 'Ada' }

function doSomethingImportant() {}
```

命名本身不是语法功能，但好的命名能降低代码理解成本。

## 5. 🤔 注释和严格模式怎么用？

ECMAScript 支持两种 C 风格注释。

单行注释使用 `//`：

```js
// 这是单行注释
const count = 10
```

块注释使用 `/* ... */`：

```js
/*
  这是块注释，
  可以跨多行。
*/
const count = 10
```

严格模式是 ES5 引入的一种更严格的解析和执行模型。它会让一些不规范、不安全或容易出错的写法直接抛错。

启用整个脚本的严格模式：

```js
'use strict'
```

也可以只在函数内部启用：

```js
function doSomething() {
  'use strict'
  // 函数体
}
```

严格模式的写法看起来像普通字符串，但 JavaScript 引擎会把它识别为指令。

## 6. 🤔 分号和代码块为什么值得认真对待？

ECMAScript 语句通常以分号结尾。分号在很多场景下可以省略，因为解析器会尝试自动补全语句结尾。

不过，实践中仍然建议保持一致的风格。无论团队选择加分号还是不加分号，都应该让格式化工具稳定处理，避免让自动分号插入成为隐蔽问题。

代码块由 `{}` 包裹，用来把多条语句组合在一起：

```js
if (enabled) {
  count += 1
  console.log(count)
}
```

虽然某些控制语句在只有一条语句时可以省略代码块，但不推荐这样写：

```js
// 不推荐
if (enabled) console.log('enabled')

// 推荐
if (enabled) {
  console.log('enabled')
}
```

始终使用代码块，能让控制结构更清楚，也能减少后续插入新语句时引入错误的概率。
