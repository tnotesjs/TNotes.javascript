## 2. 🫧 评价

操作符是 JavaScript 隐式转换最密集的区域。写业务代码时不一定要记住每条转换规则，但一定要知道哪些操作符会转数值、哪些会拼字符串、哪些会短路，以及为什么更推荐使用全等比较。

## 3. 🤔 操作符为什么容易产生意外结果？

ECMAScript 的操作符可以作用在多种类型的值上。为了让这些操作可执行，语言会在背后做类型转换。

比如：

```js
5 + '5' // '55'
5 - '2' // 3
```

同样是一个数值和一个字符串，加法会进入字符串拼接逻辑，而减法会把字符串转成数值再计算。

因此，学习操作符不能只看符号本身，还要看它面对不同类型时的求值规则。

## 4. 🤔 一元操作符有哪些常见行为？

一元操作符只操作一个值。

递增和递减操作符包括前缀版和后缀版：

```js
let age = 29
++age // 30
age-- // 29
```

前缀版会先改变值再参与表达式求值；后缀版会先使用旧值，再改变变量。

```js
let num = 2
let a = ++num // a 是 3，num 是 3
let b = num++ // b 是 3，num 是 4
```

一元加号可以触发与 `Number()` 类似的数值转换：

```js
;+'01' + // 1
  true + // 1
  'z' // NaN
```

一元减号会先转换为数值，再取负值。

## 5. 🤔 位操作符应该怎样理解？

位操作符会先把操作数转换为 32 位整数，再按二进制位进行操作，最后再转换回 JavaScript 的数值表示。

常见位操作符包括：

| 操作符 | 含义       |
| ------ | ---------- |
| `~`    | 按位非     |
| `&`    | 按位与     |
| 管道符 | 按位或     |
| `^`    | 按位异或   |
| `<<`   | 左移       |
| `>>`   | 有符号右移 |
| `>>>`  | 无符号右移 |

位操作更接近底层整数计算。它会把 `NaN` 和 `Infinity` 这类特殊值当作 `0` 处理，因此一般业务代码里不应该滥用。

它适合出现在明确需要位级计算的场景，比如权限掩码、二进制协议、图形处理等。

## 6. 🤔 布尔操作符为什么不一定返回布尔值？

布尔操作符包括逻辑非 `!`、逻辑与 `&&` 和逻辑或 `||`。

`!` 总是返回布尔值：

```js
!0 // true
!'hello' // false
!!'hello' // true
```

`&&` 和 `||` 不一定返回布尔值，它们会返回参与求值的某个操作数。

```js
'a' && 'b' // 'b'
'' || 'fallback' // 'fallback'
```

它们还有短路特征：

- `&&` 遇到第一个假值就停止。
- `||` 遇到第一个真值就停止。

这让它们常用于默认值和条件执行，但也要求你清楚参与判断的到底是真值、假值，还是严格布尔值。

## 7. 🤔 算术操作符的核心转换规则是什么？

乘法、除法、取模和指数操作符通常会把非数值操作数转换为数值。

```js
'6' * '7' // 42
'10' / 2 // 5
true % 2 // 1
```

指数操作符 `**` 等价于指数计算：

```js
3 ** 2 // 9
16 ** 0.5 // 4
```

加法比较特殊。只要有字符串参与，就容易进入字符串拼接：

```js
const num1 = 5
const num2 = 10

'sum=' + num1 + num2 // 'sum=510'
'sum=' + (num1 + num2) // 'sum=15'
```

减法没有字符串拼接语义，因此会更直接地尝试数值转换。

## 8. 🤔 关系和相等操作符为什么要谨慎使用？

关系操作符包括 `<`、`>`、`<=`、`>=`。它们比较字符串时，会比较字符编码，而不是自然语言里的字母顺序。

```js
'Brick' < 'alphabet' // true
```

如果有数值参与，另一个操作数通常会转换为数值。只要比较中出现 `NaN`，结果通常都会是 `false`。

相等操作符分两组：

| 操作符       | 是否类型转换 |
| ------------ | ------------ |
| `==`、`!=`   | 会转换       |
| `===`、`!==` | 不转换       |

示例：

```js
'55' == 55 // true
'55' === 55 // false
null == undefined // true
null === undefined // false
NaN == NaN // false
```

实践中，优先使用 `===` 和 `!==`。这样可以减少隐式转换带来的判断错误。

## 9. 🤔 条件、赋值和逗号操作符有什么用？

条件操作符是三元表达式：

```js
const max = num1 > num2 ? num1 : num2
```

它适合表达简单的二选一赋值。如果逻辑变复杂，就应该改用更清楚的 `if` 语句。

赋值操作符包括普通赋值和复合赋值：

```js
let count = 10
count += 5
count *= 2
```

复合赋值只是简写，不会提升性能。

逗号操作符可以在一条语句里执行多个表达式，并返回最后一个表达式的值：

```js
const value = (1, 2, 3)
console.log(value) // 3
```

它最常见的使用场景还是同时声明多个变量。表达式里的逗号操作符可读性通常较差，不建议频繁使用。
