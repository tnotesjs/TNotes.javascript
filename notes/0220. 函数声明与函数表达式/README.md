## 2. 🫧 评价

- 函数声明和函数表达式多数时候都能创建函数，但它们进入执行上下文的时机不同。很多“为什么这里调用不到函数”的问题，都卡在这个时机差异上。

## 3. 🤔 函数声明为什么可以先调用后定义？

因为函数声明会被提升。

JavaScript 引擎在执行代码之前，会先读取函数声明，并把函数定义放进当前执行上下文。

```js
console.log(sum(1, 2)) // 3

function sum(a, b) {
  return a + b
}
```

这段代码可以正常运行，因为 `sum` 在代码真正执行前已经完成了函数声明提升。

## 4. 🤔 函数表达式为什么不能这样用？

函数表达式只有执行到赋值语句时，变量才会保存函数对象。

```js
console.log(sum(1, 2)) // ReferenceError

const sum = function (a, b) {
  return a + b
}
```

如果使用 `var`，变量声明会提升，但赋值不会提升，因此仍然不能提前调用。

```js
console.log(sum) // undefined
console.log(sum(1, 2)) // TypeError

var sum = function (a, b) {
  return a + b
}
```

区别在于：函数声明提升的是函数定义，函数表达式提升的最多只是变量声明。

## 5. 🤔 函数声明和函数表达式语义上等价吗？

在创建出来的函数对象层面，它们很接近；但在代码执行时机上，它们不等价。

| 写法       | 是否提升函数定义 | 常见用途                             |
| ---------- | ---------------- | ------------------------------------ |
| 函数声明   | 是               | 定义可在当前作用域任意位置调用的函数 |
| 函数表达式 | 否               | 把函数当作值赋给变量、属性或参数     |

```js
function declared() {}

const expressed = function () {}
```

如果你希望函数在整个作用域内都可读可调用，函数声明更直接；如果你希望强调函数是一个值，或者需要条件赋值、传参、返回函数，函数表达式更合适。

## 6. 🤔 条件里应该写函数声明吗？

不要依赖条件块里的函数声明行为。

```js
if (condition) {
  function sayHi() {
    console.log('Hi')
  }
} else {
  function sayHi() {
    console.log('Yo')
  }
}
```

历史上不同浏览器对这种写法的处理并不一致。更稳的写法是先声明变量，再根据条件赋值函数表达式。

```js
let sayHi

if (condition) {
  sayHi = function () {
    console.log('Hi')
  }
} else {
  sayHi = function () {
    console.log('Yo')
  }
}
```

这种写法把函数作为值处理，执行路径也更明确。
