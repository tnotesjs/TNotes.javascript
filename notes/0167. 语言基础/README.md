## 3. 🤔 ECMAScript 语言核心到底包含什么？

ECMAScript 定义的是 JavaScript 的语言核心。它不负责浏览器窗口、页面节点、网络请求或文件系统，而是描述这门语言在最基本层面上怎样工作。

本章覆盖的内容可以分成五组：

| 主题     | 关注点                                        |
| -------- | --------------------------------------------- |
| 语法     | 标识符、注释、严格模式、语句结束和代码块      |
| 数据类型 | 原始类型、对象类型、类型检测和类型转换        |
| 操作符   | 数学、布尔、位、关系、相等、赋值等运算规则    |
| 语句     | 条件、循环、跳转、标签、`switch` 等流控制能力 |
| 函数     | 函数声明、参数、返回值和严格模式限制          |

这些内容看起来都是基础，但它们共同决定了 JavaScript 代码如何被解析、如何求值、如何控制执行流程。

## 4. 🤔 为什么说语言基础是后续内容的前置地图？

JavaScript 的很多复杂问题，最后都会落回语言基础。

比如：

- 变量为什么在声明前能访问到 `undefined`？这和 `var` 的声明提升有关。
- 为什么 `0.1 + 0.2` 不等于精确的 `0.3`？这和 Number 的 IEEE 754 表示有关。
- 为什么 `5 + '5'` 得到字符串，而 `5 - '2'` 得到数值？这和操作符的类型转换规则有关。
- 为什么 `switch` 里字符串 `10` 和数字 `10` 不匹配？这和 `switch` 使用全等比较有关。

也就是说，这一章不是单纯背概念，而是在建立一套解释 JavaScript 行为的基础规则。

## 5. 🤔 学这一章时最容易混淆哪些边界？

这一章最容易混淆的边界有三组。

第一组是“语言能力”和“宿主能力”。ECMAScript 规定变量、类型、操作符、语句、函数等语言能力，但不规定浏览器页面怎么显示，也不规定 `document` 或 `window` 这些对象的具体能力。

第二组是“值的类型”和“变量本身”。JavaScript 变量是松散类型的，变量并不固定绑定某种类型，真正有类型的是变量当前保存的值。

第三组是“语法合法”和“实践推荐”。比如省略分号通常合法，但不推荐；`var` 仍然合法，但现代代码更推荐 `const` 优先、`let` 次之；`with` 语句存在，但严格模式禁止，实践中也不应该使用。

## 6. 🤔 这一章应该怎样收束理解？

这一章可以收束成一句话：JavaScript 的语言核心由语法、类型、操作符、语句和函数组成，它们共同构成后续所有复杂能力的基础。

更具体地说，需要记住这些结论：

- ECMAScript 定义语言核心，不负责输入输出或浏览器 API。
- 基本数据类型包括 `Undefined`、`Null`、`Boolean`、`Number`、`String` 和 `Symbol`，复杂数据类型是 `Object`。
- JavaScript 不区分整数和浮点数，统一使用 `Number` 表示数值。
- 严格模式会限制一些容易出错或不安全的写法。
- 操作符会根据操作数类型进行不同程度的转换，其中加法、相等和关系比较最容易出意外。
- 流控制语句大多借鉴自类 C 语言，但 `for-in`、`for-of`、`switch` 等在 JavaScript 中有自己的细节。
- 函数不需要声明返回值类型，任何函数都可以在任意位置返回任意值；没有显式返回值时，结果是 `undefined`。

后面学习对象、作用域、函数、类和异步时，你会不断回到这些基础规则。它们不一定显眼，但很管用。
