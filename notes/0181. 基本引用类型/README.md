## 3. 🤔 什么叫基本引用类型？

基本引用类型可以理解为 JavaScript 运行时内置提供的一组对象模型。它们不是业务对象，而是语言和运行时为了处理常见任务预先准备好的能力。

这一章涉及的几类能力分别是：

- `Date`：处理日期、时间和时间戳。
- `RegExp`：处理模式匹配、查找和替换。
- 原始值包装类型：让字符串、数值、布尔值在读取时临时表现得像对象。
- 单例内置对象：比如 `Global`、`Math`，它们在程序开始时就已经存在。

换句话说，这一章关注的是“JavaScript 自带了哪些对象能力”，而不是“你自己怎样定义对象”。

## 4. 🤔 这一章的四类内置能力分别解决什么问题？

可以把这一章拆成四类问题来看：

| 类型         | 解决的问题                               |
| ------------ | ---------------------------------------- |
| `Date`       | 怎样表示时间、比较日期、提取年月日时分秒 |
| `RegExp`     | 怎样按模式匹配文本，而不是逐字符手写判断 |
| 包装类型     | 为什么原始值看起来也能调用方法           |
| 单例内置对象 | 哪些全局能力天生就存在，不需要实例化     |

这些能力虽然都属于“对象”，但使用方式完全不同。

比如 `Date` 和 `RegExp` 往往需要你显式创建实例；而 `Math` 不需要 `new`，直接调用它上面的属性和方法就可以。字符串包装对象则更特殊，你通常不会主动创建它，但运行时会在读取原始值时临时帮你创建。

## 5. 🤔 为什么说这一章像是在建立对象世界的入口认知？

因为从这里开始，你接触到的已经不只是“值”，而是“值背后的能力集合”。

前面几章讲变量、类型和作用域，重点还是语言基础。到了这一章，JavaScript 开始展示它内置对象体系的一部分：

- 时间不是单个数字，而是 `Date` 对象提供的一整套 API。
- 文本匹配不只是 `indexOf()`，还可以交给 `RegExp` 表达复杂规则。
- 原始值虽然不是对象，但读取时又能借到对象的方法。
- 全局上下文里已经放着一些兜底对象，供你直接调用。

这也解释了为什么后面学习集合、迭代器、类和异步 API 时，你会越来越频繁地遇到“对象方法”和“内置构造器”。

## 6. 🤔 这一章最值得优先记住哪些实践结论？

这章内容不少，但可以优先记住下面这些结论：

- `Date` 本质上是时间戳的对象化表示，比较大小时通常比较的是毫秒值。
- `RegExp` 适合做模式匹配，但越复杂的正则越要重视可读性和测试。
- 原始值能调用方法，不代表它真的变成了长期存在的对象。
- `new Boolean(false)`、`new Number(1)`、`new String('x')` 这类包装对象通常都不值得主动使用。
- `Global` 上挂的是全局能力；在浏览器里，它通常由 `window` 体现。
- `Math` 是工具对象，不是构造函数，也不应该被实例化。

## 7. 🤔 这一章应该怎样收束理解？

这一章可以收束成一句话：JavaScript 不只提供语法，还内置了一批对象能力，分别用于处理时间、文本匹配、原始值操作和全局工具函数。

更具体地说：

- 引用类型是对象能力的组织方式。
- `Date` 和 `RegExp` 是最常见的基础内置引用类型。
- 原始值包装类型解释了为什么字符串、数值和布尔值“看起来像对象”。
- `Global` 和 `Math` 则说明，有些内置能力从程序一启动就已经可用。

把这层入口认知建起来，后面再看 `Array`、`Map`、`Set`、迭代器和类时，就不会只把它们当成“某几个零散 API”。
