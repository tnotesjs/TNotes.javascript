## 3. 🤔 垃圾回收到底在做什么？

JavaScript 运行时会自动管理内存。所谓垃圾回收，本质上就是定期找出那些已经不再需要、也不再能被访问到的值，然后释放它们占用的内存。

这件事为什么重要？因为对象、数组、闭包、定时器回调都会占用内存。如果这些值已经没用了却一直保留引用，页面就可能越来越吃内存。

你可以把垃圾回收理解成一句话：不是“对象创建后多久删除”，而是“对象什么时候变得不可达”。

## 4. 🤔 标记清理和引用计数有什么区别？

现代 JavaScript 引擎主要使用标记清理。

它的大致思路是：

1. 从当前仍然可访问的值出发做标记。
2. 沿着这些值继续标记它们还能访问到的对象。
3. 没有被标记到的值，就在后续回收时清理。

另一种历史上出现过的策略是引用计数。它会记录一个值被引用了多少次；引用数为 `0` 时，就认为可以回收。

两者的核心差别在于：

- 标记清理看“现在还能不能到达这个值”。
- 引用计数看“还有多少个地方指着这个值”。

引用计数思路直观，但碰到循环引用会出问题，所以现代主流引擎早就不再把它作为核心方案。

## 5. 🤔 为什么循环引用曾经会造成问题？

如果两个对象互相引用，引用计数就可能永远不归零：

```js
function problem() {
  const objectA = {}
  const objectB = {}

  objectA.other = objectB
  objectB.other = objectA
}
```

函数执行完以后，按理说这两个对象都该没用了。但如果垃圾回收策略只看“引用数是不是 0”，那它们会因为彼此还引用着对方而留在内存里。

旧版 IE 里，DOM 对象和 JavaScript 对象的循环引用还会进一步放大这个问题。所以早年的经验经常强调一件事：不用的引用要及时断开。

```js
myObject.element = null
element.someObject = null
```

现在主流引擎已经不靠这套旧机制工作了，但“别让无意义引用长期存在”这个原则仍然成立。

## 6. 🤔 日常开发里怎样做内存管理更稳妥？

大多数时候，你不需要手动“释放内存”，但需要主动减少不必要的引用。

比较常见的做法有：

- 全局对象不再需要时，把引用设为 `null`。
- 大对象或缓存失效时，及时清空对它的持有。
- 事件监听器、定时器、订阅关系在不用时解除。
- 避免把本来只在局部用的数据挂到全局对象上。

比如：

```js
function createPerson(name) {
  return { name }
}

let globalPerson = createPerson('Nicholas')

globalPerson = null
```

这里把 `globalPerson` 设为 `null`，不是立刻删对象，而是去掉一条重要引用，让这个对象在后续垃圾回收时更容易被认定为可回收。

## 7. 🤔 哪些写法最容易导致内存泄漏？

常见内存泄漏场景主要有三类。

第一类是意外的全局变量：

```js
function setName() {
  name = 'Jake'
}
```

如果没有使用声明关键字，这个 `name` 在非严格模式下可能直接挂到全局对象上，生命周期会比你预期长很多。

第二类是一直不清理的定时器、监听器和订阅：

```js
const name = 'Jake'

const timerId = setInterval(() => {
  console.log(name)
}, 1000)

clearInterval(timerId)
```

只要回调还活着，它引用的外部变量就可能继续活着。

第三类是闭包长期持有大对象或大数据：

```js
function outer() {
  const data = { name: 'Jake' }

  return function inner() {
    return data.name
  }
}
```

只要返回的内部函数还存在，`data` 就不能被轻易回收。

## 8. 🤔 性能敏感场景下还能做什么优化？

如果代码非常在意性能，可以进一步考虑减少对象抖动，也就是不要频繁创建大量短命对象。

书里提到几个方向：

- 优先用 `const` 和 `let`，让作用域更小、更容易结束。
- 不要随意给对象动态增删属性，尽量保持对象结构稳定。
- 需要高频复用对象时，可以考虑对象池或复用已有对象。

比如，相比 `delete obj.key`，很多时候把属性设为 `null` 更稳妥：

```js
const article = {
  title: 'Inauguration Ceremony Features Kazoo Band',
  author: 'Jake',
}

article.author = null
```

这样既表达了“这里不再需要这个引用”，也通常比动态改对象结构更容易让引擎保持优化状态。
