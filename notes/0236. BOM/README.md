## 2. 🫧 评价

- BOM 是浏览器里最像工具箱的一层 API。它不直接描述文档结构，却决定了窗口、地址、浏览器信息、屏幕信息和历史记录这些真实浏览环境能力怎么被 JavaScript 使用。

## 3. 🤔 BOM 是什么？

BOM 是浏览器对象模型，英文是 `Browser Object Model`。它提供了与网页内容本身无关、但与浏览器环境密切相关的一组对象。

如果说 DOM 主要让你操作文档节点，那么 BOM 更像是在操作浏览器窗口及其周边能力，比如：

- 当前窗口和弹出窗口。
- 当前页面地址。
- 浏览器和设备信息。
- 屏幕信息。
- 浏览器历史记录。

早期 BOM 长期缺少统一规范，很多能力都是浏览器厂商各自扩展出来的。后来 HTML5 将其中大量共同行为标准化，才让 BOM 的核心能力逐渐稳定下来。

## 4. 🤔 BOM 的核心为什么是 `window`？

`window` 是 BOM 的核心对象，它代表当前浏览器窗口。

在浏览器中，`window` 有两重身份：

| 身份 | 含义 |
| --- | --- |
| 全局对象 | ECMAScript 的全局变量、全局函数和全局构造函数都可以挂在它上面 |
| 浏览器窗口接口 | 提供窗口尺寸、位置、滚动、打开新窗口、定时器和系统对话框等能力 |

```js
var age = 29

console.log(window.age) // 29
```

通过 `var` 声明的全局变量会成为 `window` 的属性。这个行为是浏览器环境中理解全局作用域的重要背景。

## 5. 🤔 本章几个 BOM 对象分别负责什么？

| 对象        | 主要职责                                                   |
| ----------- | ---------------------------------------------------------- |
| `window`    | 浏览器窗口、全局作用域、窗口关系、尺寸位置、定时器和对话框 |
| `location`  | 当前 URL 信息、地址解析和页面导航                          |
| `navigator` | 浏览器、设备、插件和部分平台能力信息                       |
| `screen`    | 屏幕尺寸、可用区域、颜色深度和方向等显示器信息             |
| `history`   | 浏览器会话历史记录、前进后退和历史状态管理                 |

这些对象通常都能通过 `window` 访问，例如 `window.location`、`window.navigator`、`window.screen` 和 `window.history`。

## 6. 🤔 学完这一章应该留下哪些结论？

可以先抓住几条主线：

- BOM 以 `window` 为基础，`window` 既是浏览器窗口对象，也是浏览器环境中的全局对象。
- `top`、`parent`、`self` 等窗口指针用于描述窗口和子窗口之间的关系。
- `location` 可以读取并修改 URL，设置 `href`、`search`、`hash` 等属性可能触发导航。
- `location.replace()` 会替换当前历史记录，用户不能通过后退按钮回到被替换页面。
- `navigator` 提供浏览器和平台信息，但这些信息存在兼容性、准确性和隐私限制。
- `screen` 主要提供显示器信息，日常业务代码中使用频率不高。
- `history` 可以在不知道具体 URL 的情况下前进、后退，也可以通过状态管理 API 支持单页应用路由。

简单来说，BOM 管的是浏览器运行环境，不是页面文档结构。写浏览器端 JavaScript 时，很多看起来理所当然的全局能力，其实都来自这一层。
