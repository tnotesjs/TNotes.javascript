## 2. 🫧 评价

- 用户代理检测像是在读一份历史悠久、充满妥协的身份证。它有用，但你必须知道它为什么长得这么乱，以及为什么不能把它当作绝对真实。

## 3. 🤔 什么是用户代理检测？

用户代理检测是通过用户代理字符串判断浏览器、系统和设备信息的方式。

在客户端 JavaScript 中，可以通过 `navigator.userAgent` 读取用户代理字符串。

```js
console.log(navigator.userAgent)
```

在服务器端，用户代理字符串通常来自 HTTP 请求头。服务器可以根据它返回不同资源或做统计分析。

在客户端，用户代理检测通常不如能力检测可靠，因此应该作为最后选择。

## 4. 🤔 用户代理字符串为什么这么复杂？

HTTP 规范原本只要求浏览器发送类似 `产品名/版本` 的简短字符串。

早期 Mosaic 的用户代理字符串就很简单：

```txt
Mosaic/0.9
```

后来 Netscape Navigator、IE、Gecko、WebKit、Chrome、Opera、移动浏览器陆续加入竞争。很多网站会根据用户代理字符串决定是否提供完整功能。

为了不被网站拒绝，后来者经常把自己伪装成已经被网站支持的浏览器。于是，用户代理字符串变得越来越长，也越来越像一层层历史兼容信息的叠加。

## 5. 🤔 为什么很多浏览器都标识自己像 Mozilla？

Netscape 的开发代号是 Mozilla，早期市场影响很大。很多服务器会检测字符串中是否包含 Mozilla，再决定是否提供更好的页面。

IE 为了兼容这些网站，在用户代理字符串中加入了 `Mozilla` 和 `compatible` 一类信息。

后来 Gecko、WebKit、Chrome 等也为了兼容既有检测逻辑，在字符串中保留了大量类似标识。

例如，现代 Chrome 的用户代理字符串里通常会同时出现 `Mozilla`、`AppleWebKit`、`KHTML, like Gecko`、`Chrome` 和 `Safari` 等片段。

这不是因为它真的是这些浏览器的简单组合，而是用户代理历史中不断兼容旧检测脚本的结果。

## 6. 🤔 用户代理检测能识别哪些信息？

如果解析规则足够新，用户代理字符串通常能推断出这些信息：

- 浏览器名称。
- 浏览器版本。
- 渲染引擎。
- 操作系统。
- 操作系统版本。
- 设备类型，例如桌面或移动设备。
- 设备厂商和型号。

这些信息对统计、日志、下载页分发、兼容性提示和少量兜底逻辑有价值。

但用户代理检测的前提是你相信这段字符串，也相信解析规则覆盖了当前浏览器和设备。

## 7. 🤔 用户代理字符串可以伪造吗？

可以。

书中提到，`navigator.userAgent` 在现代浏览器里通常是只读属性，直接赋值不会生效。

```js
navigator.userAgent = 'fake'

console.log(navigator.userAgent) // 通常仍是原值
```

但这不代表它完全不能被改写。浏览器开发工具、扩展、自动化环境、代理服务器或某些脚本技巧都可能影响用户代理字符串。

更重要的是，浏览器自己也可能出于兼容性或隐私考虑让它看起来不那么真实。

所以，如果你怀疑用户代理字符串被篡改，继续写更复杂的反篡改检测通常意义不大。更务实的做法是回到能力检测。

## 8. 🤔 应该自己手写用户代理解析器吗？

通常不建议。

用户代理字符串变化非常快。新浏览器、新版本、新操作系统、新设备会持续出现，手写解析器很容易过期。

书中提到，可以使用维护较活跃的第三方解析库，例如：

- `UAParser.js`
- `Bowser`
- `Platform.js`
- `Google Closure` 中的相关能力

这些库也需要更新，但至少维护工作由更专业的库集中处理。

如果项目只是为了判断一个具体能力，仍然不应该引入用户代理解析器，而应该直接做能力检测。

## 9. 🤔 用户代理检测什么时候还有价值？

用户代理检测适合下面这类场景：

- 统计用户浏览器、系统和设备分布。
- 服务器根据设备类型返回不同下载包。
- 已知某个浏览器版本有明确缺陷，而这个缺陷无法通过能力检测表达。
- 需要识别渲染引擎或操作系统做兼容提示。

它不适合用来决定每一处业务逻辑。

例如，判断是否支持某个 API 时，不要写：

```js
const isChrome = /Chrome\//.test(navigator.userAgent)
```

而应该写：

```js
if ('serviceWorker' in navigator) {
  // 使用 Service Worker
}
```

用户代理检测可以回答“这大概是什么环境”，能力检测更适合回答“能不能做这件事”。
