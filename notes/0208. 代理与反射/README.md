## 3. 代理与反射这一章在讲什么？

这一章讲的是 ES6 新增的元编程能力：在对象外面包一层代理，让你有机会拦截读取、赋值、枚举、删除、函数调用、构造函数调用等基本操作。

代理的基本结构有三部分：

- 目标对象：真正被代理的对象。
- 处理程序对象：用于定义拦截逻辑。
- 代理对象：外部代码实际操作的对象。

```js
const target = { name: 'Jake' }

const proxy = new Proxy(target, {
  get(target, property, receiver) {
    console.log(`读取 ${String(property)}`)
    return Reflect.get(target, property, receiver)
  },
})

console.log(proxy.name)
```

代理的价值不在于创建另一个普通对象，而在于把对象操作变成可拦截、可观察、可重写的过程。

## 4. 代理和反射是什么关系？

代理负责拦截操作，反射负责提供这些操作的默认实现。

比如你定义了 `get` 捕获器，就可以用 `Reflect.get()` 把默认读取行为转发回目标对象：

```js
const proxy = new Proxy(target, {
  get(target, property, receiver) {
    return Reflect.get(target, property, receiver)
  },
})
```

这也是 `Reflect` 在代理场景里特别常见的原因：它让你可以在默认行为前后加逻辑，而不是手写一份容易出错的底层操作。

## 6. 这一章最后应该收束成哪些结论？

可以把本章收束成四句话：

- 代理是真实对象外面的一层透明抽象，但它和目标对象不是同一个对象。
- 捕获器可以拦截大量基本操作，但必须遵守语言规定的不变式。
- 反射 API 是捕获器的天然搭档，常用于恢复或组合默认行为。
- 代理适合做访问跟踪、验证、隐藏、绑定和可观察对象，但不适合随意改变对象语义。

还要特别注意：代理属于 ES6 的基础性运行时能力，很多代理行为无法被转译成 ES5 代码。因此它更适合在明确支持 `Proxy` 和 `Reflect` 的环境中使用。
