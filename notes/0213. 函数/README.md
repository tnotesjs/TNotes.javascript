## 3. 🤔 为什么说函数也是对象？

在 ECMAScript 中，函数是 `Function` 类型的实例。既然是对象，函数就可以有属性和方法，也可以被变量引用、作为参数传递、作为返回值返回。

最常见的函数声明写法是：

```js
function sum(num1, num2) {
  return num1 + num2
}
```

函数表达式则是把一个函数对象赋值给变量：

```js
const sum = function (num1, num2) {
  return num1 + num2
}
```

箭头函数也是函数表达式的一种更简洁写法：

```js
const sum = (num1, num2) => num1 + num2
```

还有一种不推荐的写法是使用 `Function` 构造函数：

```js
const sum = new Function('num1', 'num2', 'return num1 + num2')
```

这种写法会把字符串形式的代码重新解释成函数体，性能和可维护性都不好。它更适合理解函数确实是对象，而不是日常开发中的首选写法。

## 5. 🤔 学完这一章应该留下哪些结论？

可以先记住几条稳定结论：

- 函数名本质上是保存函数对象引用的变量。
- JavaScript 函数调用不检查参数数量和类型，因此不存在传统意义上的函数重载。
- 箭头函数更适合短回调和需要词法 `this` 的场景，但不适合当构造函数。
- 函数声明会提升，函数表达式要等执行到赋值语句后才可用。
- 闭包让函数可以保留外层作用域，但也意味着相关变量会继续留在内存中。
- 私有变量和特权方法可以通过闭包模拟出来。

简单来说，函数既是 JavaScript 的执行单元，也是对象模型、作用域模型和模块化模式的共同基础。
