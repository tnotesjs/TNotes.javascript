## 2. 🫧 评价

- 函数表达式的核心价值是把函数明确当作值处理。只要你在赋值、传参、返回值里使用函数，基本就进入了函数表达式的世界。

## 3. 🤔 函数表达式是什么？

函数表达式是一个会产生函数对象的表达式，最常见写法是把函数赋值给变量。

```js
const sayHi = function () {
  console.log('Hi')
}
```

这里函数没有自己的显式名称，因此通常称为匿名函数表达式。

也可以写成命名函数表达式：

```js
const sayHi = function innerSayHi() {
  console.log('Hi')
}
```

命名函数表达式里的名称主要在函数内部和调试栈中有用。

## 4. 🤔 函数表达式会提升吗？

函数表达式不会像函数声明那样提升函数定义。

```js
sayHi() // ReferenceError

const sayHi = function () {
  console.log('Hi')
}
```

只有执行到赋值语句之后，变量才会保存函数对象。

这也是函数表达式和函数声明最关键的差别之一。

## 5. 🤔 条件里为什么更适合用函数表达式？

当函数实现需要根据条件变化时，先声明变量，再用函数表达式赋值更明确。

```js
let sayHi

if (condition) {
  sayHi = function () {
    console.log('Hi')
  }
} else {
  sayHi = function () {
    console.log('Yo')
  }
}
```

这比在条件块里写两个同名函数声明更稳，因为历史上不同环境对块级函数声明的处理并不完全一致。

## 6. 🤔 函数表达式为什么常用于返回函数？

因为函数表达式本质是值，所以它很自然适合被返回。

```js
function createGreeting(prefix) {
  return function (name) {
    return `${prefix}, ${name}`
  }
}

const sayHello = createGreeting('Hello')

console.log(sayHello('Jake')) // Hello, Jake
```

这个模式常用于创建定制函数，也常和闭包一起出现。

## 7. 🤔 什么时候应该优先考虑函数表达式？

以下场景更适合函数表达式：

- 需要把函数赋给变量或对象属性。
- 需要根据条件选择不同函数实现。
- 需要把函数作为参数传递。
- 需要从另一个函数中返回函数。
- 需要用命名函数表达式稳定引用自身。

简单来说，当你不是单纯声明一个可提升的函数，而是在“操作一个函数值”时，函数表达式通常更贴切。
