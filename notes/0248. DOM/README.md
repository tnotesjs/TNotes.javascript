## 3. DOM 是什么？

DOM 是 Document Object Model 的缩写，中文通常叫文档对象模型。它把 HTML 或 XML 文档表示成一棵由节点组成的树，并提供一组可以用 JavaScript 操作这棵树的 API。

浏览器加载页面后，会把文档解析成 DOM 树。页面中的元素、文本、注释、文档类型声明等内容都会成为不同类型的节点。

你通过 DOM 可以做这些事：

- 读取页面结构。
- 查找某个元素。
- 修改元素属性和文本。
- 创建、插入、移动、删除节点。
- 动态加载脚本和样式。
- 观察 DOM 变化。

简单来说，DOM 是 JavaScript 和页面文档之间的桥。

## 4. DOM 为什么用节点树表示文档？

HTML 文档天然具有嵌套关系。

```html
<html>
  <body>
    <main>
      <p>Hello DOM</p>
    </main>
  </body>
</html>
```

这类结构很适合用树来表示：文档是根，元素是分支，文本是叶子。DOM 的节点层级让你可以从任意节点出发，向上找到父节点，向下访问子节点，或在同一层访问兄弟节点。

这套模型的关键在于：DOM 不是字符串模板，而是运行时对象模型。你修改节点，浏览器会根据修改更新页面状态。

## 5. 本章围绕哪些主线展开？

本章可以分成三条主线。

第一条是节点层级。它介绍 `Node`、`Document`、`Element`、`Text`、`Comment`、`DocumentFragment` 等节点类型，以及它们之间的关系和常用操作方法。

第二条是 DOM 编程。它关注动态脚本、动态样式、表格操作，以及 `NodeList`、`HTMLCollection` 这类实时集合带来的性能和遍历问题。

第三条是 `MutationObserver`。它用于观察 DOM 的属性、子节点和文本变化，用异步回调替代早期性能较差的 DOM 变更事件。

## 6. DOM 操作最容易忽略什么？

最容易忽略的是 DOM 操作的成本和副作用。

DOM API 看起来像普通对象操作，但它背后连接着浏览器的解析、样式、布局和渲染系统。频繁查询和修改 DOM，可能触发额外计算。

另外，很多 DOM 集合是实时的。例如 `childNodes`、`getElementsByTagName()` 返回的集合会随着文档变化自动更新。遍历这类集合时，如果同时修改 DOM，很容易写出意料之外的循环。

## 7. 学完这一章应该留下哪些结论？

这一章的核心结论可以整理为几条：

- `Node` 是 DOM 节点类型的基础，很多节点关系和操作方法都来自它。
- `Document` 表示整个文档，`Element` 表示元素，文本、注释、文档类型、文档片段等也都有对应节点类型。
- 创建节点只是创建对象，插入文档后才会影响页面。
- `DocumentFragment` 适合临时承载一组节点，插入时只转移它的子节点。
- 动态脚本和动态样式可以通过 DOM 创建，但它们有加载、执行和兼容性细节。
- DOM 集合经常是实时集合，遍历时要注意缓存长度或转换为普通数组。
- `MutationObserver` 以异步方式收集 DOM 变化记录，适合观察结构、属性和文本变更。

DOM 编程的底层思路是：把页面当作一棵对象树来理解，同时尊重这棵树和渲染引擎之间的连接。
