## 3. 🤔 JavaScript API 解决了什么问题？

ECMAScript 规定了语言本身，但浏览器里的 JavaScript 不只是运行语法。现代 Web 应用需要访问文件、处理二进制数据、播放媒体、拖放内容、显示通知、感知页面可见性、流式处理数据、测量性能、封装组件和完成加密操作。

这些能力都不是语言核心的一部分，而是浏览器平台通过各种 API 暴露给 JavaScript 的能力。

简单来说，本章讨论的是 JavaScript 如何从“操作页面”继续扩展为“使用浏览器平台”。它涉及的 API 很散，但都围绕一个共同目标：让 Web 应用拥有更接近桌面应用的能力，同时仍然受浏览器安全模型约束。

## 4. 🤔 这些 API 可以分成哪些能力域？

本章可以按能力域理解，而不是把所有 API 当成彼此孤立的知识点。

| 能力域 | 相关笔记 | 典型问题 |
| --- | --- | --- |
| 并发与内存 | `Atomics`、`SharedArrayBuffer` | 多个执行上下文如何共享内存并避免竞争。 |
| 跨上下文通信 | `postMessage()` | 页面、窗口、iframe 如何安全传递消息。 |
| 文本与二进制 | Encoding API、File API、Blob API、Streams API | 字符串、字节、文件和流如何互相转换。 |
| 媒体与交互 | 媒体元素、原生拖放、Notifications API、Page Visibility API | 如何播放媒体、拖放数据、提醒用户和暂停后台任务。 |
| 性能与组件 | 计时 API、Web 组件 | 如何测量性能、封装可复用 UI。 |
| 安全与加密 | Web Cryptography API | 如何生成安全随机数、摘要、密钥、签名和加密数据。 |

这样拆分后会更容易发现关联：例如 Encoding API 经常和 Streams API、File API 一起使用；Page Visibility API 经常和媒体播放、轮询、动画调度配合；Web Crypto 基本围绕 `ArrayBuffer` 和定型数组工作。

## 5. 🤔 哪些 API 和安全权限有关？

很多浏览器 API 不能随便调用，因为它们会触及隐私、安全或系统资源。

典型例子包括：

- `SharedArrayBuffer` 现代浏览器通常要求跨源隔离环境，避免高精度计时侧信道攻击。
- Notifications API 需要用户授权，且权限请求通常应该由用户手势触发。
- File API 只能读取用户通过文件选择框或拖放显式提供的文件，不能任意访问本地文件系统。
- Web Cryptography API 中的 `crypto.subtle` 要求安全上下文，通常也就是 HTTPS。
- 跨上下文消息必须校验 `origin`，否则可能接收来自不可信页面的数据。

所以，使用这些 API 时不能只问“浏览器支不支持”，还要问“当前上下文允不允许”。

## 6. 🤔 哪些 API 适合处理二进制和流式数据？

本章里有一组 API 都在处理字符串、字节和数据流之间的转换：

- Encoding API：在字符串和 `Uint8Array` 之间转换。
- File API 与 Blob API：表示用户选择的文件和不可变二进制数据。
- Streams API：按块读取、写入和转换数据。
- Web Cryptography API：很多输入输出都使用 `ArrayBuffer` 或定型数组。

这组 API 的共同点是，它们不再把数据看作普通字符串，而是把数据看作字节序列。这对文件上传、下载处理、流式解码、摘要计算、加密解密都很重要。

## 7. 🤔 哪些 API 直接影响用户体验？

媒体元素、原生拖放、Notifications API 和 Page Visibility API 都直接影响用户体验。

媒体元素让浏览器原生支持音频和视频播放，并允许开发者自定义控制器。原生拖放让用户可以用熟悉的拖拽方式移动文本、链接、图片或文件。Notifications API 可以在页面外显示通知，但必须尊重用户授权。Page Visibility API 让应用知道页面是否处于后台，从而暂停轮询、动画或媒体播放。

这些 API 都有一个共同点：用户体验好坏取决于边界处理。比如自动播放策略、拖放目标默认行为、通知权限请求时机、页面隐藏后的任务暂停，都需要仔细设计。

## 8. 🤔 本章小结要抓住哪些结论？

这一章展示的是现代浏览器平台的扩展能力。JavaScript 可以通过标准 API 访问共享内存、跨上下文通信、文本编码、文件和二进制数据、媒体播放、拖放、通知、页面可见性、流、性能计时、组件封装和加密能力。

这些 API 的成熟度并不完全相同。有些能力已经非常常见，例如 `postMessage()`、`Blob`、`FileReader`、`performance.now()`；有些能力更依赖环境、权限或兼容性，例如 `SharedArrayBuffer`、Notifications API、Web Crypto、Web Components 的部分细节。

学习本章时，与其死记 API 表，不如抓住三条线：数据如何表示，能力如何授权，状态如何变化。把这三条线串起来，浏览器 API 就不再是一堆零散接口，而是 Web 平台逐步开放能力的路线图。
