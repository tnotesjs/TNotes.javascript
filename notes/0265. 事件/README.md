## 3. 🤔 什么是事件？

事件表示文档或浏览器窗口中发生的某个有意义的时刻。用户点击按钮、页面加载完成、输入框获得焦点、窗口尺寸变化、设备方向变化，这些都可以触发事件。

JavaScript 通过事件处理程序响应这些时刻。

```js
const button = document.querySelector('button')

button.addEventListener('click', () => {
  console.log('按钮被点击')
})
```

这种模式本质上接近观察者模式：代码先订阅某类事件，事件发生时浏览器再调用对应函数。

## 4. 🤔 事件为什么是 JavaScript 与 HTML 交互的核心？

HTML 负责结构，CSS 负责表现，JavaScript 负责行为。事件把用户动作和 JavaScript 行为连接起来。

没有事件，页面只能静态展示；有了事件，页面才能响应输入、更新状态、提交数据、打开弹窗、拖拽元素、处理键盘快捷键和监听设备变化。

好的事件代码通常有两个特点：

- 只在需要时响应事件。
- 响应逻辑和页面结构保持适度解耦。

这也是现代开发中更推荐 `addEventListener()`、事件委托和组件化事件管理的原因。

## 6. 🤔 事件流要解决什么问题？

当你点击一个按钮时，按钮当然被点击了；但它所在的父元素、页面主体、文档甚至窗口，也都可以认为接收到了这次点击。

事件流描述的就是事件在 DOM 树中的传播顺序。

DOM 事件流通常分为三阶段：

1. 捕获阶段：从外层向目标元素传播。
2. 到达目标阶段：事件到达真正触发的元素。
3. 冒泡阶段：从目标元素向外层祖先传播。

实际开发中，冒泡阶段最常用，因为它非常适合事件委托。

## 7. 🤔 事件处理程序有哪些注册方式？

历史上有几种注册事件处理程序的方式。

| 方式          | 示例                        | 现代建议     |
| ------------- | --------------------------- | ------------ |
| HTML 事件属性 | 在标签上写事件属性          | 不推荐       |
| DOM0          | `element.onclick = handler` | 简单场景可见 |
| DOM2          | `addEventListener()`        | 推荐         |
| 旧 IE         | `attachEvent()`             | 历史兼容背景 |

现代代码通常优先使用 `addEventListener()`。

```js
element.addEventListener('click', handleClick)
element.removeEventListener('click', handleClick)
```

它支持同一事件绑定多个处理程序，也能选择捕获阶段或冒泡阶段。

## 8. 🤔 事件对象提供了什么？

事件发生时，浏览器会创建事件对象，把本次事件的相关信息放进去。

```js
element.addEventListener('click', (event) => {
  console.log(event.type)
  console.log(event.target)
  console.log(event.currentTarget)
})
```

常见信息包括：

- 事件类型。
- 真正触发事件的目标。
- 当前正在执行处理程序的元素。
- 鼠标坐标。
- 键盘按键。
- 是否可取消默认行为。
- 当前传播阶段。

事件对象还提供 `preventDefault()` 和 `stopPropagation()`，分别用于阻止默认行为和停止传播。

## 9. 🤔 事件类型为什么这么多？

浏览器事件体系覆盖了很多输入和状态变化。

常见类别包括：

- UI 事件，例如 `load`、`resize`、`scroll`。
- 焦点事件，例如 `focus`、`blur`、`focusin`、`focusout`。
- 鼠标和滚轮事件，例如 `click`、`mousemove`、`wheel`。
- 键盘与输入事件，例如 `keydown`、`keyup`、`input`。
- 合成事件，例如输入法组合输入相关事件。
- HTML5 事件，例如 `DOMContentLoaded`、`hashchange`、`pageshow`。
- 设备事件，例如方向和运动事件。
- 触摸和手势事件。

不需要机械记住所有事件名。更重要的是理解高频事件的触发时机、事件对象信息和性能边界。

## 10. 🤔 事件代码为什么要关注性能和内存？

事件处理程序本质上是函数对象。绑定得越多，占用内存越多，页面初始化和交互成本也可能越高。

此外，如果 DOM 节点被移除，但它的事件处理程序和外部引用没有清理，旧浏览器和复杂页面中可能出现内存无法释放的问题。

因此，本章强调两点：

- 用事件委托减少监听器数量。
- 移除 DOM 或不再需要监听时，及时移除处理程序。

现代框架通常会帮你处理一部分清理工作，但理解底层规则仍然很重要。

## 11. 🤔 模拟事件有什么用？

模拟事件是用 JavaScript 创建事件对象，再主动派发到某个目标上。

```js
const event = new Event('change', { bubbles: true })
input.dispatchEvent(event)
```

它常用于：

- 测试交互逻辑。
- 触发自定义组件行为。
- 把程序状态变化转换成事件通知。
- 派发自定义事件。

模拟事件会进入正常事件流，触发对应监听器。但它不等于真实用户操作，也不一定触发浏览器默认行为。

## 12. 🤔 学完这一章应该留下哪些结论？

本章可以收束为几条结论：

- 事件是 JavaScript 与页面交互的主要机制。
- DOM 事件流包含捕获、目标和冒泡三个阶段。
- 现代事件注册优先使用 `addEventListener()`。
- 事件对象提供目标、阶段、默认行为和传播控制等上下文信息。
- 事件类型非常多，学习重点应放在高频事件和行为边界上。
- 事件委托能显著减少监听器数量，也能适应动态 DOM。
- 不再需要的事件处理程序应该移除。
- 模拟事件适合测试和组件通信，但不能完全替代真实用户交互。

事件代码写得好，页面交互会清晰、稳定、可维护；事件代码失控，页面会很快变成一堆彼此牵扯的回调。
