## 2. 🫧 评价

TC39 与 ECMA-262 => 这两个术语紧密关联，核心都指向 => 「JavaScript 语言的标准化」。

- TC39 是一群制定 JS 语言标准的人
- ECMA-262 是 JS 语言标准

## 3. 🤔 ECMA-262 是什么？

ECMA-262 是由 「Ecma International」（一个国际标准组织）发布的标准文档，它正式定义了 ECMAScript 这门语言的规范。JavaScript 是 ECMAScript 最广为人知的实现，其他实现还包括 JScript、ActionScript 等。

简单说：ECMA-262 就是 JavaScript 的「语法规则手册」，规定了语法、类型、语句、运算符、内置对象等一切语言层面的细节。

## 4. 🤔 TC39 是什么？

TC39（Technical Committee 39）是 Ecma International 下设的第 39 号技术委员会，负责「维护和演进 ECMA-262 标准」。成员包括来自各大浏览器厂商（Google、Mozilla、Apple、Microsoft 等）和主要科技公司的代表。

### 4.1. 提案流程（TC39 Process）

TC39 以「提案（Proposal）」的方式推动语言演进，每个提案经历五个阶段：

| 阶段    | 含义                                                 |
| ------- | ---------------------------------------------------- |
| Stage 0 | 纳粹/想法阶段，仅作为 strawman 提交                  |
| Stage 1 | 正式提案，开始描述问题和高层 API 设计                |
| Stage 2 | 草案，初步语法和语义已确定                           |
| Stage 3 | 候选，规范文本完成，等待实现和反馈                   |
| Stage 4 | 完成，已有两个以上独立实现通过验收测试，将被纳入标准 |

每年发布一版新的 ECMAScript 标准（如 ES2015/ES6、ES2024 等），包含所有在该周期内达到 Stage 4 的提案。

### 4.2. 一个重要的里程碑：ES2015（ES6）

- ES2015（ES6） 是一个里程碑，引入了 `let/const`、箭头函数、模块、Promise、解构等大量特性。
- 此后每年迭代，持续为语言加入新能力，如 `async/await`、可选链 `?.`、空值合并 `??`、Temporal API 等。
- 所有主流引擎（V8、SpiderMonkey、JavaScriptCore）都以此规范为实现依据，确保跨平台一致性。

如果你想跟踪最新提案进展，可以直接访问 [tc39/proposals][1] 的 GitHub 仓库。

## 5. 🤔 谁会去看标准？

标准文档非常晦涩，不是给普通人看的。主要的读者群体：

### 5.1. 引擎开发者

这是最核心的受众。V8（Chrome/Node.js）、SpiderMonkey（Firefox）、JavaScriptCore（Safari）这些 JavaScript 引擎的工程师，需要严格依照标准来实现语言行为。对他们来说，标准就是"需求文档"。

### 5.2. 工具链/编译器作者

Babel、TypeScript、SWC、ESLint 等工具的开发者，需要精确理解语法语义才能正确地解析、转译或检查代码。比如 Babel 要把新语法转换成旧语法，就得知道新语法到底怎么解析、怎么求值。

### 5.3. 框架/库的资深开发者

当遇到边界情况（edge case）或者微妙行为时，去查 MDN 发现文档说得不够详细，就会直接翻标准找确切定义。比如：

```js
[] + [] // ""
[] + {} // "[object Object]"
{} + [] // 0
```

这类让人困惑的结果，标准里都有精确的类型转换规则。

### 5.4. 语言设计者和研究者

想了解一门语言为什么这样设计、参与语言演进讨论的人。

### 5.5. 普通开发者怎么看

说实话，大多数开发者「不需要」直接读标准。更实际的选择是：

- MDN — 面向开发者的权威参考，比标准易读得多
- tc39/proposals — 了解新特性，看提案的 README 和示例就够
- 语言规范的通俗解读 — 比如《你不知道的 JavaScript》、各种博客文章

在日常开发中，大部分时候看一些类似 MDN 这样的技术文档就足够了，不需要去看 [ECMA-262 Standard][2]。

### 5.6. 标准的最大价值

标准的最大价值在于「最终裁决」 => 当有争议或不确定时，它是唯一的权威答案。比如你在看一些技术文章时发现说法不一致，或者某个行为让你困惑，这时候直接去看标准，查找最权威的答案。
